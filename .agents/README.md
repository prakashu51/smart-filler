# Agents Workspace

This folder tracks implementation planning and iteration artifacts for the extension.

## Purpose

- Keep a lightweight history of what changed in each iteration
- Capture architecture decisions before code shifts
- Make future phases easier to execute without losing context

## Suggested Structure

- `iterations/`: one markdown note per implementation iteration
- `decisions/`: architecture decisions and tradeoffs
- `plans/`: phase-level plans and execution breakdowns

## Naming Convention

- `iteration-001-<topic>.md`
- `decision-001-<topic>.md`
- `phase-01-<topic>.md`

## Working Rule

After each meaningful iteration, add a short note covering:

- what changed
- why it changed
- risks or follow-ups
- next step
